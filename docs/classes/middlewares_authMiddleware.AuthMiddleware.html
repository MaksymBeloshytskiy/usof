  • isAuthenticated(req, res, next): Promise<void>

  • Middleware to check if the user is authenticated.

    This middleware verifies the presence and validity of the authorization token in the request headers. It checks if the token is blacklisted, verifies the token, and attaches the decoded user information to the request object.

StaticisPostAuthorOrAdmin

  • isPostAuthorOrAdmin(req, res, next): Promise<void>

  • Middleware to check if the current user is the author of the post or an admin.

    This middleware function checks if the user making the request is either the author of the post specified in the request parameters or has an admin role. If the user is authorized, the request is passed to the next middleware or route handler. Otherwise, an appropriate error response is sent.

    Parameters

    • req: CustomRequest

      The request object, expected to contain the current user and post ID in the parameters.

    • res: Response<any, Record<string, any>>

      The response object used to send error responses if the user is not authorized.

    • next: NextFunction

      The next middleware or route handler to be called if the user is authorized.

    Returns Promise<void>

    Throws

    If the current user is not available in the request.

    Throws

    If the post ID is missing in the request parameters or the post is not found.

    Throws

    If the user is neither the author of the post nor an admin.

    Throws

    If any other known error occurs during authorization.

    Throws
